How to set up a subversion server for Repos

Version control in Repos and Project Engine relies on Subversion.

Subversion is open source (free to install), and widely supported. There are at least 10 books on Amazon about it, and a responsive mailing list as well as the free Subversion book. Contrary to other document management systems, Repos does not lock you in to a proprietary storage solution. If you no longer need Repos or Project Engine, you still have all your data and history in a standard Subversion repository, which will always be free of charge. You can use clients such as Tortoise SVN to access your files.

Repos provides hosting at a monthly fee, but during 2006 this service is only available in Sweden. But it can also be installed on top of your own Subversion server.

Installing Subversion

Three steps needed to set up a server for version control:

  1. Find a suitable server and install Apache2
  2. Install Subversion
  3. Create a repository and test it

The procedure requires basic server and network administration skills. The instructions below only outline the process. The details depend on the operating system. The goal for the server administrator is to get Subversion connected to Apache.

Find a suitable server and install Apache2

Most computers on your local network can work as a subversion server. Apache and Subversion can be installed on Windows, Mac, Unix and Linux. In most Linux distribution, Subversion installation is straightforward using the package manager. One advantage with Linux over Windows is that SSL (secure transfer) support is standard. SSL encrypts your passwords and transfered data when the repository is used from outside your firewall or internet router.

On Windows, we recommend purchasing a configured server bundle from Repos. For windows servers that do not have Apache as web server, this makes at least 20 hours of hassle with getting compiled and compatible Apache, PHP, Subversion and OpenSSL. It includes the installation support you need to get your repository up and running.
