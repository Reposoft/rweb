Javascript framework

The repos javascript framework.

Objectives

Our requirements specification:

  1. repos.se should be fully accessible without a single javascript.
  2. Every page has it's own URL, relating to the main contents item of the page.
  3. The user can bookmark contents.
  4. Formatting is done with CSS.
  5. Javascript is used for adding behaviour to the page.

Releases contain only head.js, with compressed scripts.

tests

Selenium TestSuite. Can be started via the repos testing startpage.

This contains only tests for the platform functionality. Plugins have their own tests.

head.js

All pages import /repos/scripts/head.js. It is composed by scripts from this folder that have been accepted for release. When head.js file is modified, the cache headers should make sure that all browsers get the new script package. Those headers are tested in HTTP headers test.

tools

Some of the tests print to the FireBug console.

For the default firefox script console, we recommend Firefox JavaScript console sidebar.
