Repos installation

Server setup checklist

  1. Create a subversion repository. See the Subversion book.
  2. Set up subversion with apache and the user database. You can get some help from the repos admin section.
  3. Check that you can use the repository without repos. Try to log in from a browser to the empty repository root. If you're on windows, use the TortoiseSVN client to check out the repository.
  4. Install the repos www folder in a /repos directory at your server root. This must be at the same hostname and port as for direct repository access, because browsers don't like cross-domain references in scripts and stylesheets.
  5. Configure repos according to the configuration instructions
  6. Add the SVNIndexXSLT directive to the apache repository configuraion, pointing to /repos/view/repos.xsl. If the /repos folder is configured correctly the basic functionality works.
  7. Some external resources can not be included in the distribution due to license matters. There is ant scripts to download those. Examples are /smarty and themes/simple/icons. Install Apache Ant, look for build.xml files and run the command ant in the sam folder.
  8. Copy the client certificate to the server root, so that it can be installed into users' browsers. Run ant in the conf/svn-config-dir/ folder to install server settings.
  9. Optional components are the installed individually. For example, to enable repos Quay, install it into a /quay folder at the server root.

Att installera i organisationen

Repos upptäcks i tre steg. I det första agerar Optime data bara konsult vid installation av beprövad servermjukvara (open source). Ingen hemsnickrad kod alltså. Plattformen installeras på befintliga Windows, Linux eller OS X servrar, parallellt med nuvarande system. Redan då har ni möjliggjort flexiblare fildelning, ett enkelt Webbgränssnitt för att komma åt allt innehåll, samt versionshantering för befintligt Intranät.

Efter detta steg kan man utvärdera, komma fram till att plattformen inte gör den nytta jag utlovar, och avbryta utan att ha förbrukat nämnvärt med tid eller pengar.

Steg två är att att lägga på användbarhet i webbgrässnittet, samt att införa versionshantering i det dagliga arbetet. Exempel på funktioner är delad kalender och att flera personer enkelt kan ändra i samma dokument, med en ständigt uppdaterad senaste version.

Steg tre är att vidareutveckla ert intranät, exemplevis med webbaserad projekthantering. Det är först nu som det behövs utvecklingstid. Men plattformen gör nu att applikationen blir tunn (måttlig mängd kod mellan användare och lagring). Allt kan integreras med klientprogramvara, oavsett operativsystem. Filarkivet fungerar även som en vanlig mapp i Windows. Outlook har utbyte med webbkalendern och andra kalenderprogram.
