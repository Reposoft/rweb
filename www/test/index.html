Automated test suites

Reset the test repository before running automated tests.

The "run" links will execute the test suite in the current browser and post the results

Repository tests require manual login as user 'test'.

  • Browser test for new users
  • Configuration test run
  • Functional testing run
  • Admin functional test admin
  • Unit tests (server side) run
  • Javascripts unit tests run

There are also Performance tests

Note that tests assume the web application to be located at [current host]/repos/.

Some test cases reveal server details. They only modify the test repository, but it could be a good idea to restrict access to this folder to internal hosts.
