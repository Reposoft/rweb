Check the HTTP headers of a file from mod_dav_svn
open /repos/test/headers/
clickAndWait repositorypublicfile
verifyTextPresent |HTTP/1.1 200 OK|
verifyTextPresent |Content-Type: text/xml|
verifyTextPresent |Last-Modified:
verifyTextPresent |Content-Length:
verifyTextPresent |ETag:
verifyTextNotPresent |Cache-Control:

It is important that browsers reload the directory listings after an svn operation. The size of the xml is usually small, so it is not a problem if we get many reloads. There has been some discussion on this in the SVN list, see for example Last-Modified HTTP header from mod_dav_svn.
